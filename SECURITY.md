# 보안

Marqo는 소프트웨어 제품과 서비스의 보안을 중요하게 다룹니다.

Marqo 저장소에서 보안 취약점을 발견했다고 생각한다면 아래 절차에 따라 제보해 주세요.

## 지원 버전

보안 업데이트는 최신 Marqo 버전을 기준으로 지원합니다.

## 보안 이슈 제보

공개 GitHub 이슈로 보안 취약점을 제보하지 마세요.

대신 `support@marqo.ai`로 이메일을 보내 주세요.

가능한 범위에서 아래 정보를 포함하면 문제의 성격과 영향을 더 빠르게 파악하는 데 도움이 됩니다.

- 이슈 유형: 버퍼 오버플로, SQL 인젝션, 크로스 사이트 스크립팅 등
- 문제가 드러나는 소스 파일의 전체 경로
- 영향을 받는 소스 코드 위치: 태그, 브랜치, 커밋 또는 직접 URL
- 재현에 필요한 특수 설정
- 단계별 재현 방법
- 가능한 경우 proof-of-concept 또는 exploit 코드
- 공격자가 이 문제를 어떻게 악용할 수 있는지를 포함한 영향 범위

이 정보는 제보를 더 빠르게 분류하고 대응하는 데 사용됩니다.

제보 후 72시간 안에 답변을 드립니다. 문제가 확인되면 복잡도에 따라 가능한 한 빠르게 패치를 배포합니다.

## 선호 언어

공식 보안 커뮤니케이션은 영어를 선호합니다. 이 저장소의 문서는 한국어로 제공하지만, 보안 제보는 영어로 작성하면 대응이 더 원활합니다.
