# 말해진 모든 것을 인덱싱하기 - 녹음된 음성 속 정보 검색

이 문서는 오디오 파일을 전사하고 Marqo에 인덱싱해, 텍스트 질의로 관련 음성 구간을 찾는 방법을 설명합니다. 목표는 녹음된 회의, 강의, 팟캐스트, 영상 음성처럼 사람이 직접 훑기 어려운 자료를 검색 가능한 데이터로 바꾸는 것입니다.

## 오디오 작업의 어려움

오디오는 텍스트보다 검색하기 어렵습니다. 사용자는 특정 문장이나 주제를 찾고 싶지만, 원본 오디오는 시간 축 위에 놓여 있어 임의 접근이 쉽지 않습니다. 또한 여러 화자가 섞이거나 배경 소음이 있으면 전사 품질과 검색 품질 모두 영향을 받습니다.

## 음성 데이터 검색의 활용

- 회의 녹취에서 특정 결정 사항 찾기
- 강의나 팟캐스트에서 주제별 구간 찾기
- 고객 상담 기록 분석
- 영상/오디오 자료 기반 Q&A 만들기

## 오디오와 Marqo

Marqo는 오디오 자체를 직접 검색하기보다는, 전사된 텍스트와 시간 메타데이터를 인덱싱하는 방식으로 활용합니다. 각 문서는 음성의 한 구간을 나타내며, 텍스트, 시작/종료 시간, 화자 정보, 파일 경로를 포함할 수 있습니다.

## 수집

처리할 오디오 파일을 다운로드하거나 로컬 폴더에 준비합니다. 예제 스크립트는 단계별로 다운로드, 처리, 검색, 채팅 Q&A를 나누어 실행합니다.

```bash
python '1. Download.py'
```

## 데이터

각 오디오 파일은 전사 후 여러 조각으로 나뉩니다. 조각을 너무 짧게 만들면 문맥이 부족하고, 너무 길게 만들면 검색 결과가 뭉뚱그려질 수 있습니다. 사용 사례에 맞게 chunk 크기를 조절하세요.

## 화자 분리와 음성 인식

PyAnnote 같은 도구로 화자를 분리하고, Whisper 또는 다른 speech-to-text 모델로 텍스트를 만듭니다. Hugging Face API 키와 OpenAI API 키가 필요한 단계가 있을 수 있습니다.

```bash
python '2. Process.py'
```

이 단계는 GPU 사용을 권장합니다.

## 인덱싱

전사 결과를 Marqo 문서로 변환합니다.

```python
documents = [
    {
        "_id": "audio-1-0001",
        "text": "The extracted transcript segment goes here.",
        "speaker": "speaker_0",
        "start": 12.4,
        "end": 18.9,
        "audio_file": "episode.wav",
    }
]

mq.index("speech-search").add_documents(
    documents,
    tensor_fields=["text"],
)
```

## 검색

대화형 프롬프트에서 질문을 입력하면 관련 전사 구간을 반환합니다.

```bash
python '3a. Search.py'
```

## 대화형 답변 생성

검색 결과를 LLM 컨텍스트로 넣으면 자연어 답변을 만들 수 있습니다.

```bash
python '3b. chat.py'
```

이 구조는 RAG와 같습니다. Marqo는 근거 구간을 검색하고, 생성 모델은 그 근거를 사용해 답변합니다.

## 한계

- 전사 오류가 있으면 검색 결과도 영향을 받습니다.
- 화자 분리가 완벽하지 않을 수 있습니다.
- 긴 오디오에서는 chunking 전략이 품질에 큰 영향을 줍니다.
- 민감한 음성 데이터는 저장, 전사, API 전송 시 보안과 개인정보 보호를 고려해야 합니다.

## 마무리

음성 데이터는 전사와 메타데이터를 결합하면 검색 가능한 지식 자산이 됩니다. Marqo는 전사 조각을 빠르게 검색하고, 관련 오디오 구간을 찾거나 Q&A 컨텍스트로 활용하는 데 적합합니다.
