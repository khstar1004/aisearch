# Marqo 기반 2차 유료 고도화 제안 기능

본 문서는 `docs/plan.md`의 1차 개발 범위 이후, **LLM 없이 Marqo 검색엔진 기능과 애플리케이션 로직만으로 추가 제안할 수 있는 유료 고도화 항목**을 정리한다.

1차 개발은 텍스트 검색, 이미지 검색, 텍스트+이미지 혼합 검색, Top 3 추천, 카테고리 추천, 관련 상품 리스트, MSSQL 동기화, JS 위젯을 구축하는 범위다.

2차 고도화는 이미 구축된 Marqo 인덱스와 검색 로그를 활용해 **검색 정확도, 상품 탐색, 대체상품 추천, 운영 리포트, 가맹점 전환율 개선**을 상품화하는 방향이 적합하다.

---

## 1. 전제

## 1.1 LLM은 제외한다

이 문서의 기능은 ChatGPT 같은 LLM을 붙이지 않는 것을 전제로 한다.

따라서 아래 기능은 제외한다.

```
- 자연어 상담 챗봇
- 추천 사유 문장 생성
- 견적 안내문 자동 작성
- 고객 문의 답변 자동 생성
- 업종/행사 조건을 문장으로 해석하는 AI 상담
```

대신 Marqo가 제공하는 검색 기능과 서버 로직으로 구현 가능한 기능만 다룬다.

## 1.2 활용 가능한 Marqo 기능

2차 고도화에서 활용할 수 있는 Marqo 기능은 다음과 같다.

| Marqo 기능 | 활용 방향 |
| --- | --- |
| Tensor search | 의미 기반 상품 검색 |
| Image search | 이미지 유사 상품 검색 |
| Weighted query | 텍스트+이미지 가중치 조합 |
| Searchable attributes | 상품명, 카테고리, 설명, 이미지 등 필드별 검색 |
| Lexical search | 정확 키워드, 상품번호, 브랜드, 규격 검색 보강 |
| Hybrid search | AI 의미 검색 + 키워드 검색 결합 |
| Filter | 가격, 카테고리, 판매상태, mall_id 조건 검색 |
| Facets | 카테고리/가격대/속성별 결과 집계 |
| Recommend | 특정 상품 기준 유사 상품 추천 |
| Score modifiers | 판매량, 클릭률, 최신성, 마진 등 비즈니스 점수 반영 |
| Sort / rerank | 가격순, 최신순, 인기순, 추천순 노출 |
| Collapse | 같은 상품군/중복 상품 묶기 |
| Custom vector | 추후 외부 임베딩 모델 교체 실험 |

## 3. 과금 후보 1: Hybrid 검색 정확도 고도화

## 3.1 제안명

```
AI 검색 정확도 고도화 패키지
```

## 3.2 현재 1차 기능과의 차이

1차 검색은 주로 Marqo의 `TENSOR` 검색을 중심으로 동작한다. 이는 의미 기반 검색에는 강하지만, 판촉물 쇼핑몰에서 자주 발생하는 아래 검색에는 약할 수 있다.

```
- 상품번호 검색
- 브랜드명 검색
- 모델명 검색
- 500ml, 3단, 2단, A4, 16GB 같은 규격 검색
- UV, 스텐, 무광, 고급형 같은 짧은 키워드
- 정확한 카테고리명 검색
```

2차에서는 Marqo의 `HYBRID` 또는 `LEXICAL` 검색을 함께 적용해 의미 검색과 정확 키워드 검색을 결합한다.

## 3.3 제공 기능

```
- 텍스트 검색을 Tensor 단독에서 Hybrid 검색으로 고도화
- 상품명/카테고리/키워드/설명 필드별 검색 가중치 조정
- 상품번호/규격/브랜드 검색 정확도 보강
- 한글/영문 혼용 키워드 검색 품질 개선
- 검색어별 Tensor/Hybrid 결과 비교 리포트 제공
```

## 3.4 클라이언트 설득 포인트

```
1차 AI 검색은 의미 기반으로 상품을 찾아주는 기능입니다.
2차 정확도 고도화는 판촉물 쇼핑몰에서 실제로 많이 쓰는 상품번호, 규격, 브랜드, 가격 조건 검색까지 안정적으로 잡아주는 작업입니다.
검색 결과가 좋아지면 고객 이탈과 상담 문의 부담이 줄어듭니다.
```

## 3.5 구현 개요

```
- Marqo searchMethod HYBRID 적용
- lexical 검색 대상 필드 지정
- tensor 검색 대상 필드 지정
- 검색어 유형별 전략 분기
- 검색 결과 score 비교
- 기존 검색 로그 기반 품질 평가
```

## 4. 과금 후보 2: 비슷한 상품/대체상품 추천

## 4.1 제안명

```
AI 대체상품 추천 기능
```

## 4.2 기능 설명

상품 상세 페이지 또는 검색 결과에서 특정 상품을 기준으로 비슷한 상품을 추천한다.

판촉물 구매자는 보통 한 상품만 보고 바로 결정하지 않는다. 가격, 납기, 인쇄 방식, 수량, 디자인을 비교한다. 따라서 “비슷하지만 더 저렴한 상품”, “같은 용도의 고급형 상품”, “같은 카테고리 인기상품” 추천은 상담과 전환에 직접 도움이 된다.

## 4.3 제공 기능

```
- 이 상품과 비슷한 상품 추천
- 더 저렴한 대체상품 추천
- 더 고급형 대체상품 추천
- 같은 카테고리 인기상품 추천
- 이미지가 유사한 상품 추천
- 텍스트 속성이 유사한 상품 추천
- 품절/비노출 상품의 대체상품 추천
```

## 4.4 화면 적용 위치

```
- 상품 상세 페이지 하단
- 검색 결과 카드의 "비슷한 상품" 버튼
- 품절/비노출 상품 페이지
- 장바구니 또는 견적 문의 전 단계
```

## 4.5 Marqo 활용 방식

```
- Marqo recommend API 또는 특정 상품 text/image 기반 재검색
- category_name filter 적용
- price range filter 적용
- status active filter 적용
- mall_id 정책 반영
```

## 4.6 클라이언트 설득 포인트

```
고객이 마음에 드는 상품을 찾았지만 가격, 납기, 디자인 때문에 망설일 때 바로 대체상품을 보여줄 수 있습니다.
상담원이 수동으로 비슷한 상품을 찾는 시간을 줄이고, 고객이 사이트 안에서 더 오래 비교하게 만들 수 있습니다.
```

## 5. 과금 후보 3: 가격대/수량대 기반 추천 필터

## 5.1 제안명

```
예산 맞춤 AI 상품 추천
```

## 5.2 기능 설명

판촉물 구매는 예산과 수량이 중요하다. 1차 검색이 “검은 우산”을 찾는 기능이라면, 2차는 “개당 3,000원 이하 검은 우산”, “500개 제작에 적합한 상품”처럼 구매 조건에 맞춰 결과를 좁혀준다.

LLM 없이도 UI에서 조건을 구조화하면 구현 가능하다.

## 5.3 제공 기능

```
- 가격대 필터: 1,000원 이하, 3,000원 이하, 5,000원 이하, 1만원 이상
- 수량대 필터: 100개, 300개, 500개, 1,000개 이상
- 예산 기반 필터: 총예산 / 수량으로 단가 범위 계산
- 카테고리 + 가격대 조합 검색
- 저가형/일반형/고급형 추천 탭
```

## 5.4 Marqo 활용 방식

```
- price 필드 filter
- category_name filter
- score modifier로 가격대 선호 반영
- sort by price 옵션 제공
```

## 5.5 클라이언트 설득 포인트

```
판촉물 고객은 예산 안에서 상품을 찾는 경우가 많습니다.
예산/수량 조건을 검색에 직접 붙이면 고객이 상담 전 스스로 후보 상품을 좁힐 수 있고, 가맹점 상담 효율도 올라갑니다.
```

## 6. 과금 후보 4: 카테고리 Facet 검색 고도화

## 6.1 제안명

```
AI 카테고리 탐색 고도화
```

## 6.2 현재 1차 기능과의 차이

1차 기능의 비슷한 카테고리 추천은 검색 결과를 애플리케이션에서 후처리해 만든다.

2차에서는 Marqo facet 기능을 활용해 검색 결과의 카테고리, 가격대, 속성 분포를 더 안정적으로 집계하고, 쇼핑몰 탐색 UI에 붙인다.

## 6.3 제공 기능

```
- 검색 결과 기준 카테고리별 상품 수 표시
- 가격대별 상품 수 표시
- 인기 카테고리 우선 노출
- 카테고리 클릭 시 AI 검색 조건 유지
- 모바일 필터 UI 제공
```

## 6.4 예시

검색어:

```
검은색 선물용
```

결과 필터:

```
우산 124개
텀블러 86개
볼펜 73개
가방 48개
상패 21개
```

## 6.5 클라이언트 설득 포인트

```
검색 결과가 많을 때 고객이 카테고리로 빠르게 좁혀볼 수 있습니다.
단순 추천 태그보다 쇼핑몰 필터에 가까운 기능이라 실제 구매 탐색에 더 유리합니다.
```

## 7. 과금 후보 5: 클릭/인기 기반 랭킹 고도화

## 7.1 제안명

```
검색 전환율 개선 랭킹 패키지
```

## 7.2 기능 설명

1차 검색은 AI 유사도 중심으로 정렬한다. 하지만 실제 운영에서는 유사도만 높은 상품보다 클릭, 문의, 판매 가능성이 높은 상품을 위에 올리는 것이 중요하다.

2차에서는 검색/클릭 로그를 기반으로 상품별 인기 점수를 만들고, Marqo score modifier 또는 애플리케이션 rerank 로직으로 결과 순위를 조정한다.

## 7.3 제공 기능

```
- 검색 결과 클릭률 기반 인기 점수 계산
- 많이 클릭된 상품 우선 노출
- 최근 많이 찾는 상품 가중치
- 신상품 가중치
- 재고/판매가능 상품 가중치
- 가맹점별 인기상품 가중치
- 무클릭 상품 하향 조정
```

## 7.4 Marqo 활용 방식

```
- popularity_score 필드 추가
- click_score 필드 추가
- updated_at 기반 recency score 반영
- score modifiers 또는 서버 후처리 rerank 적용
```

## 7.5 클라이언트 설득 포인트

```
AI가 비슷한 상품을 찾는 것에서 끝나는 것이 아니라, 실제 고객이 많이 클릭하고 구매 가능성이 높은 상품을 더 위에 보여줍니다.
검색 품질을 매출 관점으로 개선하는 기능입니다.
```

## 8. 과금 후보 6: 가맹점별 추천 최적화

## 8.1 제안명

```
1,700개 가맹점 맞춤 AI 검색 최적화
```

## 8.2 기능 설명

해오름기프트는 다수 가맹점이 같은 DB를 공유한다. 1차에서는 `mall_id`를 받아 URL/가격/노출 정책을 반영한다.

2차에서는 가맹점별 검색/클릭 데이터를 반영해 사이트마다 조금 다른 추천 결과를 제공한다.

## 8.3 제공 기능

```
- 가맹점별 인기 검색어 집계
- 가맹점별 많이 클릭된 상품 우선 노출
- 특정 가맹점 제외 상품/카테고리 강화
- 가맹점별 대표 카테고리 우선 추천
- 가맹점별 가격 정책 반영 랭킹
- 가맹점별 검색 품질 리포트
```

## 8.4 Marqo 활용 방식

```
- mall_id 필터
- mall별 popularity_score 별도 관리
- 검색 로그 기반 score field 업데이트
- 공통 인덱스 + mall 정책 후처리
```

## 8.5 클라이언트 설득 포인트

```
모든 가맹점에 같은 AI 검색을 제공하는 것에서 한 단계 더 나아가, 각 가맹점의 고객 행동에 맞춘 검색 결과를 제공할 수 있습니다.
가맹점 수가 많은 해오름 구조에서 차별화 포인트가 됩니다.
```

## 9. 과금 후보 7: 무결과/저품질 검색어 리포트

## 9.1 제안명

```
AI 검색 품질 운영 리포트
```

## 9.2 기능 설명

검색 기능은 배포 후 관리가 중요하다. 고객이 무엇을 찾았는데 못 찾았는지, 어떤 검색어에서 결과 품질이 낮은지 알아야 상품명, 키워드, 카테고리, 이미지 품질을 개선할 수 있다.

현재 1차 구현에도 검색 로그와 품질 인사이트 기반이 있으므로, 2차에서는 이를 운영 리포트와 관리자 화면으로 상품화한다.

## 9.3 제공 기능

```
- 무결과 검색어 목록
- 낮은 유사도 검색어 목록
- 반복 검색되지만 클릭 없는 검색어
- 많이 검색되는 키워드
- 많이 클릭되는 상품
- 이미지 검색 실패/저품질 이미지 통계
- 카테고리별 검색 수요 변화
- 동의어/키워드 보강 후보 추출
```

## 9.4 Marqo 활용 방식

```
- 검색 결과 score 활용
- 검색/클릭 로그 기반 분석
- score threshold 기반 low-confidence 집계
- 검색어별 top result 추적
```

## 9.5 클라이언트 설득 포인트

```
AI 검색은 설치 후 끝나는 기능이 아니라 운영하면서 좋아지는 기능입니다.
무결과 검색어와 낮은 품질 검색어를 리포트로 제공하면, 상품 DB와 키워드 품질 개선에 바로 사용할 수 있습니다.
```

## 10. 과금 후보 8: 상품 상세 페이지 추천 영역

## 10.1 제안명

```
상품 상세 AI 추천 위젯
```

## 10.2 기능 설명

검색 팝업뿐 아니라 기존 상품 상세 페이지에 AI 추천 영역을 추가한다.

고객이 특정 상품을 보고 있을 때 아래 추천을 자동으로 보여준다.

```
- 이 상품과 비슷한 상품
- 더 저렴한 상품
- 같은 카테고리 인기상품
- 함께 비교할 만한 상품
- 이미지가 유사한 상품
```

## 10.3 화면 적용 예시

```
[상품 상세 정보]

AI가 찾은 비슷한 상품
[상품 A] [상품 B] [상품 C] [상품 D]

더 저렴한 대체상품
[상품 E] [상품 F] [상품 G] [상품 H]
```

## 10.4 Marqo 활용 방식

```
- 현재 상품 document id 기반 recommend
- 현재 상품 이미지 기반 image similarity search
- category_name filter
- price filter
- status active filter
```

## 10.5 클라이언트 설득 포인트

```
검색을 사용하지 않는 고객에게도 AI 추천을 노출할 수 있습니다.
상품 상세 페이지에서 이탈하기 전에 대체 후보를 보여주므로 체류 시간과 문의 가능성을 높일 수 있습니다.

## 12. 과금 후보 10: 중복/유사 상품 묶기

## 12.1 제안명

```
AI 유사 상품 정리 및 중복 노출 개선
```

## 12.2 기능 설명

상품 수가 많으면 같은 상품이 색상, 옵션, 공급사, 이미지 차이로 여러 번 노출될 수 있다. 검색 결과에 비슷한 상품이 과도하게 반복되면 고객은 선택하기 어려워진다.

2차에서는 Marqo similarity와 collapse/rerank 로직을 활용해 검색 결과 다양성을 개선한다.

## 12.3 제공 기능

```
- 같은 상품군 중복 노출 완화
- 동일 이미지/유사 이미지 상품 그룹 후보 추출
- 검색 결과에서 같은 카테고리 과밀 노출 완화
- 대표 상품 우선 노출
- 색상/가격/옵션이 다른 상품은 비교 가능하게 묶기
```

## 12.4 Marqo 활용 방식

```
- image similarity score 기반 유사 상품 그룹 추출
- product_name similarity 기반 후보 추출
- collapse 또는 서버 후처리
- 대표상품 score field 적용
```

## 12.5 클라이언트 설득 포인트

```
상품 수가 많을수록 검색 결과의 다양성과 정리가 중요합니다.
비슷한 상품이 반복 노출되는 문제를 줄이면 고객이 더 빠르게 비교하고 선택할 수 있습니다.
```

AI 이미지 검색 품질 개선 패키지
```

## 13.2 기능 설명

1차 이미지 검색은 대표 이미지 URL과 사용자 업로드 이미지를 기준으로 유사 상품을 찾는다.

2차에서는 이미지 검색 품질을 높이기 위해 검색 대상 이미지와 점수 계산 방식을 개선한다.

- custom vector 필드 실험 가능

## 14.5 클라이언트 설득 포인트

```
AI 검색 품질 개선은 감으로 판단하면 안 됩니다.
실제 상품 데이터와 대표 검색어로 모델별 결과를 비교해, 재색인이나 모델 교체가 필요한지 근거를 제공합니다.
```

## 15. 패키지화 제안

## 15.1 2차 기본 고도화 패키지

가장 먼저 제안하기 좋은 범위.

```
1. Hybrid 검색 정확도 고도화
2. 비슷한 상품/대체상품 추천
3. 가격대/카테고리 필터 추천
4. 무결과/저품질 검색어 리포트
```

```
1차는 AI 검색 기능 구축입니다.
2차는 실제 운영 데이터를 기반으로 검색 정확도와 구매 전환율을 높이는 고도화입니다.
```

## 15.2 2차 확장 패키지

기본 고도화 이후 제안.

```
1. 클릭/인기 기반 랭킹
2. 상품 상세 AI 추천 위젯
3. 가맹점별 추천 최적화
4. 시즌/행사별 추천관
```

판매 포인트:

```
검색창을 쓰는 고객뿐 아니라 상품 상세 페이지, 기획전, 가맹점별 운영 화면까지 AI 추천을 확장합니다.
```

## 15.3 3차 품질 운영 패키지

장기 운영 계약 또는 월 유지보수로 제안.

```
1. 이미지 검색 품질 개선
2. 중복/유사 상품 정리
3. 모델/인덱스 품질 비교 리포트
4. 월간 검색 품질 리포트
```

판매 포인트:

```
AI 검색은 한 번 설치하고 끝나는 기능이 아니라, 검색 로그와 상품 데이터를 기반으로 계속 좋아지는 운영형 기능입니다.
```

## 16. 클라이언트 제안용 한 문단

```
1차 개발은 텍스트, 이미지, 텍스트+이미지 기반으로 상품을 찾아주는 AI 검색 구축입니다.
2차 고도화는 별도의 LLM 없이 현재 구축한 Marqo 검색엔진 기능만으로 검색 정확도를 높이고, 비슷한 상품/대체상품 추천, 예산·가격대 필터, 클릭 기반 인기 랭킹, 가맹점별 추천 최적화까지 확장하는 작업입니다.
해오름기프트는 상품 수가 많고 가맹점 사이트가 많기 때문에, 단순히 검색창을 하나 추가하는 것보다 고객이 더 빨리 비교하고 선택하게 만드는 추천/랭킹 고도화가 실제 매출과 상담 효율 개선에 더 직접적으로 기여할 수 있습니다.
```

## 17. 우선 제안 순서

클라이언트에게 한 번에 모든 기능을 제안하기보다 아래 순서가 적합하다.

```
1. 검색 정확도 고도화
   - Hybrid 검색
   - 상품번호/규격/브랜드 검색 보강

2. 대체상품 추천
   - 상품 상세/검색 결과에서 비슷한 상품 추천
   - 더 저렴한 상품, 고급형 상품 추천

3. 운영 리포트
   - 무결과 검색어
   - 낮은 유사도 검색어
   - 많이 클릭된 상품

4. 전환율 개선 랭킹
   - 클릭/인기/신상품/가격 정책 반영

5. 가맹점별 최적화
   - mall_id별 인기상품, 검색어, 노출 정책 반영
```

이 순서가 좋은 이유는 다음과 같다.

```
- 1번은 1차 검색 품질의 자연스러운 연장이다.
- 2번은 고객 화면에서 바로 보이는 추가 기능이다.
- 3번은 운영 개선 근거를 만든다.
- 4번은 검색 기능을 매출 개선 기능으로 포지셔닝한다.
- 5번은 해오름기프트의 1,700개 가맹점 구조에 맞춘 차별화 기능이다.
```
