## 1. 개발 목적

해오름기프트 기존 쇼핑몰에 **AI 기반 통합 상품 검색 기능**을 구축한다.

본 기능은 사용자가 입력한 **텍스트**, **이미지**, 또는 **텍스트와 이미지를 함께 입력한 조건**을 기반으로 해오름기프트 상품 DB 내에서 가장 유사한 상품을 검색·추천하는 기능이다.

세종기프트의 AI 이미지 검색 기능을 참고하되, 해오름기프트는 단순 이미지 유사 검색을 넘어 다음 기능을 포함한다.

```
1. 텍스트 기반 AI 상품 검색
2. 이미지 기반 유사 상품 검색
3. 텍스트 + 이미지 혼합 검색
4. 상위 유사 상품 3개 추천
5. 비슷한 카테고리 추천
6. 관련 상품 리스트 추천
7. 1,700개 가맹점 사이트 공통 사용 가능 구조
```

---

## 2. 개발 방향

본 기능은 기존 쇼핑몰 DB를 직접 변경하지 않고, 별도의 **AI 검색 서버**를 구축하여 연동한다.

```
기존 MSSQL 상품 DB
→ 상품 데이터 조회
→ 이미지 서버에서 상품 이미지 수집
→ AI 검색엔진에 상품 색인
→ 기존 웹사이트에서 AI 검색 API 호출
→ 검색 결과 화면 출력
```

기존 DB는 원본 데이터로 유지하고, AI 검색엔진은 별도 검색 인덱스로 운영한다.

## 3. 시스템 구성

## 3.1 전체 구조

```
[기존 해오름기프트 웹사이트 / 1,700개 가맹점 사이트]
        ↓
[AI 검색 위젯 또는 검색 API 호출]
        ↓
[AI 검색 API 서버]
        ↓
[AI 검색엔진 / Marqo 또는 유사 멀티모달 검색엔진]
        ↑
[상품 동기화 프로그램]
        ↑
[MSSQL 상품 DB + 별도 이미지 서버]
```

## 3.2 주요 구성 요소

| 구성 요소 | 역할 |
| --- | --- |
| 기존 MSSQL DB | 상품 원본 데이터 저장 |
| 이미지 서버 | 상품 대표 이미지 저장 |
| AI 검색 API 서버 | 텍스트/이미지 검색 요청 처리 |
| AI 검색엔진 | 텍스트·이미지 벡터 검색 처리 |
| 동기화 프로그램 | MSSQL 상품 데이터를 AI 검색엔진에 반영 |
| JS 위젯 | 기존 웹사이트에 삽입되는 검색 UI |

- 클릭 시 AI 검색 팝업 또는 AI 검색 영역 오픈
- PC/모바일 반응형 지원
```

## UI 예시

```
[검색어 입력창] [돋보기 아이콘] [카메라 아이콘]
```

카메라 아이콘 클릭 시 AI 검색 팝업이 열린다.

# 4.3 텍스트 기반 AI 검색

사용자가 텍스트만 입력해도 AI 검색이 수행되어야 한다.

## 예시 검색어

```
검은 우산
스텐 텀블러
점착 메모지
탁상 달력
크리스탈 상패
고급 볼펜
친환경 가방
```

## 검색 기준

```
- 상품명
- 카테고리명
- 상품 키워드
- 상품 설명
- 유사 의미어
- AI 의미 기반 유사도
```

## 결과 예시

```
검색어: “검은 우산”

결과:
- 검정 3단 자동 우산
- 고급 장우산
- UV 차단 우산
- 판촉용 우산
```

## 요구사항

```
- 단순 키워드 일치뿐 아니라 의미 기반 검색 지원
- 오타 또는 유사 표현에 대한 검색 품질 고려
- 카테고리/판매상태 필터 적용
- 검색 결과에 유사도 또는 관련도 점수 반영
```

# 4.4 이미지 기반 유사 상품 검색

사용자가 이미지만 업로드한 경우, AI가 이미지의 형태·색상·구성 요소를 분석하여 유사 상품을 검색한다.

## 처리 흐름

```
사용자 이미지 업로드
→ 이미지 전처리
→ AI 이미지 임베딩 생성
→ 상품 이미지 인덱스와 유사도 비교
→ 유사 상품 결과 반환
```

## 예시

```
사용자 업로드 이미지: 점착 메모 사진
결과:
- 방문형 점착메모
- 안내 메모지
- 메모지 케이스
```

## 요구사항

```
- 이미지 기반 유사 상품 검색
- 유사도 점수 반환
- 상위 유사 상품 3개 별도 노출
- 추가 유사 상품 리스트 노출
- 검색 결과가 지나치게 낮은 경우 안내 문구 제공
```

# 4.5 텍스트 + 이미지 혼합 검색

본 프로젝트의 핵심 고도화 기능이다.

사용자가 이미지와 텍스트를 함께 입력하면, AI 검색엔진은 두 조건을 동시에 반영하여 결과를 반환해야 한다.

## 예시 1

```
이미지: 텀블러 사진
텍스트: 검은색

검색 결과:
- 검은색 텀블러
- 블랙 보온병
- 검정 스테인리스 보틀
```

## 예시 2

```
이미지: 우산 사진
텍스트: 3단 자동

검색 결과:
- 3단 자동 우산
- 접이식 자동 우산
- 판촉용 3단 우산
```

## 예시 3

```
이미지: 상패 사진
텍스트: 크리스탈

검색 결과:
- 크리스탈 상패
- 투명 감사패
- 유리 트로피
```

## 요구사항

```
- 이미지 유사도와 텍스트 의미 유사도를 함께 반영
- 이미지와 텍스트 중 하나만 입력되어도 검색 가능
- 둘 다 입력된 경우 혼합 검색으로 처리
- 검색 결과에는 최종 유사도 점수 표시
- 향후 텍스트/이미지 가중치 조정 가능 구조
```

## 검색 가중치 예시

```
이미지만 검색: 이미지 유사도 100%
텍스트만 검색: 텍스트 관련도 100%
이미지 + 텍스트 검색: 이미지 유사도 60% + 텍스트 관련도 40%
```

실제 가중치는 PoC 결과에 따라 조정한다.

## 5. 검색 결과 화면 요구사항

# 5.1 상위 유사 상품 3개

검색 결과 상단에 AI가 판단한 **상위 유사 상품 3개**를 노출한다.

## 표시 정보

| 항목 | 내용 |
| --- | --- |
| 상품 이미지 | 대표 이미지 |
| 유사도 | 예: 유사도 76.1% |
| 상품번호 | 기존 상품번호 |
| 상품명 | 상품명 |
| 가격 | 판매가 또는 견적문의상품 |
| 상세 링크 | 클릭 시 상품 상세 페이지 이동 |

## 화면 문구 예시

```
상위 유사 상품 3개
```

## 요구사항

```
- 검색 결과 중 상위 3개 상품을 별도 영역에 노출
- 유사도는 % 형태로 표시
- 상품 클릭 시 기존 상품 상세 페이지로 이동
- 판매중/노출중 상품만 표시
- 결과가 부족한 경우 1~2개만 노출 가능
```

## 6. 데이터 연동 요구사항

# 6.1 기존 DB 연동

기존 MSSQL DB는 원본 데이터로 유지하며, AI 검색용 데이터는 별도 검색 인덱스에 구축한다.

## 필요 데이터

```
product_id
product_name
price
category_name
main_image_url
product_url
status
updated_at
is_deleted 또는 display_yn
mall_id 또는 가맹점 식별값, 필요한 경우
```

## 요구사항

```
- MSSQL DB는 read-only 권한으로 접근
- 기존 DB에 쓰기 작업 금지
- 가능하면 AI 검색용 View 생성
- 상품 변경 여부 확인을 위해 updated_at 필요
- 삭제/비노출/품절 상태값 정의 필요
```

---

# 6.2 이미지 서버 연동

상품 대표 이미지는 별도 이미지 서버에서 가져온다.

## 요구사항

```
- AI 검색 서버에서 이미지 URL 접근 가능해야 함
- 대표 이미지 URL 제공 필요
- JPG, PNG, WEBP 등 이미지 포맷 확인
- 대량 이미지 다운로드 시 서버 부하 방지 필요
- 이미지 다운로드 실패 시 로그 저장
```

## 이미지 전처리

```
- 이미지 리사이즈
- 포맷 변환
- 손상 이미지 제외
- 너무 작은 이미지 제외
- 배경/워터마크 품질 이슈 기록
```

# 6.3 상품 데이터 동기화

상품은 계속 추가/수정/삭제되므로 AI 검색 인덱스도 주기적으로 갱신되어야 한다.

## 동기화 방식

```
초기 전체 색인
→ 주기적 변경 상품 동기화
→ 삭제/비노출 상품 정리
```

## 권장 동기화 정책

| 구분 | 내용 |
| --- | --- |
| 최초 색인 | 전체 상품 1회 색인 |
| 정기 동기화 | 1시간 단위 변경 상품 반영 |
| 전체 검증 | 매일 새벽 전체 상태 검증 |
| 수동 재색인 | 특정 상품 또는 전체 상품 재색인 기능 |

## 처리 방식

```
신규 상품: AI 검색 인덱스에 추가
수정 상품: 상품명/가격/이미지 변경 반영
이미지 변경 상품: 이미지 재분석 후 재색인
삭제/비노출 상품: 검색 결과에서 제외
```

```
POST /api/ai-search
```

## 요청 방식 1: 텍스트 검색

```
{
  "mall_id":"shop001",
  "q":"검은 우산",
  "limit":20
}

```
{
  "mall_id":"shop001",
  "image_base64":"base64_encoded_image",
  "limit":20

```
multipart/form-data
- image: 업로드 이미지 파일

## 9. AI 검색엔진 요구사항

AI 검색엔진은 텍스트와 이미지를 모두 처리할 수 있는 멀티모달 검색엔진을 사용한다.

후보:

```
- Marqo
- Typesense
- Qdrant + OpenCLIP
```

본 프로젝트에서는 텍스트·이미지 혼합 검색 고도화를 고려하여 **Marqo 또는 동등한 멀티모달 검색엔진**을 우선 검토한다.

## 요구사항

```
- 텍스트 임베딩 생성 가능
- 이미지 임베딩 생성 가능
- 텍스트 + 이미지 혼합 검색 가능
- 상품 메타데이터 저장 가능
- 상품 상태/카테고리 필터 가능
- 검색 결과 유사도 점수 반환 가능
- 향후 모델 교체 가능 구조
```

## 모델 고도화 요구사항

```
- 초기 모델로 PoC 진행
- 검색 품질이 부족할 경우 상품 검색 특화 모델 검토
- 모델 교체 시 전체 상품 재색인 가능 구조 필요
- 검색엔진 교체 가능성을 고려한 추상화 구조 필요
```

## 15. 인수 기준

본 기능은 아래 기준을 만족하면 1차 구축 완료로 본다.

```
1. 텍스트 검색이 정상 동작한다.
2. 이미지 업로드 검색이 정상 동작한다.
3. 텍스트 + 이미지 혼합 검색이 정상 동작한다.
4. 검색 결과 상위 3개 상품이 별도 노출된다.
5. 관련 상품 리스트가 출력된다.
6. 비슷한 카테고리 추천이 출력된다.
7. 상품 클릭 시 기존 상품 상세 페이지로 이동한다.
8. 판매중/노출중 상품만 검색 결과에 표시된다.
9. 상품 변경 사항이 정기 동기화로 반영된다.
10. 대표 가맹점 사이트에서 정상 동작한다.
```

## 16. 최종 개발 요약

본 프로젝트는 단순 이미지 검색 기능이 아니라 **텍스트와 이미지를 함께 이해하는 AI 상품 검색 시스템**이다.

최종 개발 방향은 다음과 같다.

```
기존 MSSQL DB는 그대로 유지
→ 별도 AI 검색 인덱스 구축
→ 텍스트/이미지/혼합 검색 지원
→ 기존 웹사이트에는 AI 검색 버튼 또는 JS 위젯 삽입
→ 1,700개 가맹점 사이트에서 공통 사용 가능하도록 API화
```

핵심 개발 범위는 다음 4가지다.

```
1. AI 멀티모달 검색엔진 구축
2. MSSQL 상품 DB 및 이미지 서버 동기화
3. 텍스트·이미지 통합 검색 API 개발
4. 기존 사이트 삽입형 검색 UI 개발
```
